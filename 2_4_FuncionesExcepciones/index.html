    Funciones y manejo de excepciones

    Una funciﾃｳn en Python es como una pequeﾃｱa herramienta que realiza una tarea especﾃｭfica. En lugar de repetir el mismo bloque de cﾃｳdigo varias veces, podemos escribirlo una sola vez dentro de una funciﾃｳn y luego llamarla cada vez que la necesitemos. Esto hace que el cﾃｳdigo sea mﾃ｡s limpio, mﾃ｡s fﾃ｡cil de entender y mucho mﾃ｡s sencillo de mantener. Por ejemplo, si queremos saludar a varias personas, en lugar de escribir print("Hola, Sergio"), print("Hola, Lucﾃｭa"), etc., podemos crear una funciﾃｳn que reciba el nombre como entrada y se encargue de mostrar el saludo. Asﾃｭ, cada vez que queramos saludar a alguien, solo tenemos que llamar a esa funciﾃｳn con el nombre correspondiente.

    Ademﾃ｡s de aprender a crear funciones, en esta sesiﾃｳn aprovechamos para introducir el manejo de errores mediante excepciones. En muchos programas, pueden surgir situaciones inesperadas: por ejemplo, intentar dividir entre cero, acceder a un dato que no existe o introducir un valor no vﾃ｡lido. Las excepciones permiten detectar esos errores y responder de forma controlada, evitando que el programa se detenga bruscamente. Aprender a usar try, except y raise nos ayuda a escribir cﾃｳdigo mﾃ｡s robusto y profesional, preparado para funcionar correctamente incluso cuando algo no sale como se esperaba.

    Asi pues, lo contenidos que vamos a abordar en esta secciﾃｳn son:

    • Definiciﾃｳn de funciones con def.
    • Parﾃ｡metros posicionales, nombrados y con valores por defecto.
    • ﾃ［bito de variables: local y global.
    • Uso de return para devolver resultados.
    • Documentaciﾃｳn interna con docstrings (""" """).
    • Manejo de errores con try, except, finally, raise.

    Definiciﾃｳn bﾃ｡sica de una funciﾃｳn

    Las funciones permiten dividir un programa en bloques lﾃｳgicos que realizan tareas especﾃｭficas. Esto mejora la legibilidad, facilita la reutilizaciﾃｳn del cﾃｳdigo y permite detectar errores mﾃ｡s fﾃ｡cilmente. En proyectos reales, como los desarrollos en Odoo, la modularidad es clave para mantener el cﾃｳdigo organizado y escalable.

    La sintaxis en la definiciﾃｳn de una funciﾃｳn en Python viene determinada por def

    Ejemplo de definiciﾃｳn de una funciﾃｳn saludar con un parﾃ｡metro

    def saludar(nombre):
    """Muestra un saludo personalizado."""
    print(f"Hola, {nombre}")

    Esta funciﾃｳn recibe un argumento (nombre) y muestra un mensaje. El docstring explica brevemente quﾃｩ hace la funciﾃｳn.

    Parﾃ｡metros por defecto y nombrados

    Los parﾃ｡metros por defecto permiten definir valores que se usarﾃ｡n si no se especifican en la llamada. Para definir un parﾃ｡metro por defecto, basta con asignar un valor mediante un = en la cabecera de la funciﾃｳn

    Ejemplo de funciﾃｳn con parﾃ｡metro por defecto

    def saludar(nombre="invitado"):
    print(f"Hola, {nombre}")

saludar()          # Hola, invitado

    Tambiﾃｩn se pueden usar argumentos nombrados para mayor claridad:

    saludar(nombre="Lucﾃｭa")

    Ejemplo de funciﾃｳn con argumentos nombrados (y con valores por defecto)

    def saludar( nombre = "Sergio", apellido = "Rey" ):

    En Python, no puedes definir un valor por defecto para un parﾃ｡metro si hay otro parﾃ｡metro sin valor por defecto despuﾃｩs de ﾃｩl. Es decir, los parﾃ｡metros con valores por defecto deben ir al final de la lista de parﾃ｡metros.

    Por ejemplo, esto no es vﾃ｡lido:

    Incorrecto

    pass

    Python lanzarﾃ｡ un SyntaxError porque b no tiene valor por defecto, pero estﾃ｡ entre dos parﾃ｡metros que sﾃｭ lo tienen.

    En cambio, esto sﾃｭ es vﾃ｡lido:

    Correcto

    pass

    Si necesitas flexibilidad en el orden, puedes usar argumentos nombrados al llamar a la funciﾃｳn, o bien reestructurar la definiciﾃｳn para que los parﾃ｡metros sin valor por defecto estﾃｩn primero.

    La palabra clave pass

    En Python, la palabra clave pass se utiliza como una instrucciﾃｳn vacﾃｭa. Sirve para indicar que no se va a ejecutar ninguna acciﾃｳn en ese lugar del cﾃｳdigo, pero que la estructura estﾃ｡ ahﾃｭ por motivos de sintaxis o planificaciﾃｳn.

    Por ejemplo, si estﾃ｡s definiendo una funciﾃｳn pero aﾃｺn no has decidido quﾃｩ va a hacer, puedes escribir:

    pass

    Esto evita que el intﾃｩrprete lance un error por tener una funciﾃｳn sin contenido. Lo mismo ocurre con clases, bucles o condicionales. Es como decir: 窶彗quﾃｭ irﾃ｡ algo mﾃ｡s adelante, pero por ahora dﾃｩjalo en blanco窶.

    Tambiﾃｩn se usa cuando necesitas que el programa siga funcionando sin hacer nada en ciertos casos:

    if letra == "h":
        pass  # No hacemos nada con la letra 'h'
    else:
        print(letra)

    Es ﾃｺtil para mantener la estructura del cﾃｳdigo mientras lo desarrollas o cuando quieres ignorar ciertos casos sin romper la lﾃｳgica del programa.

    Funciones con nﾃｺmero de parﾃ｡metros variables

    En Python, a veces necesitamos definir funciones que puedan recibir una cantidad variable de argumentos, sin saber cuﾃ｡ntos serﾃ｡n exactamente. Esto es ﾃｺtil cuando queremos que la funciﾃｳn sea flexible y se adapte a diferentes situaciones sin tener que modificar su definiciﾃｳn.

    Para ello, Python ofrece dos mecanismos:

    • *args: permite recibir cualquier nﾃｺmero de argumentos posicionales.
    • **kwargs: permite recibir cualquier nﾃｺmero de argumentos nombrados (clave-valor).

    Tambiﾃｩn se pueden combinar ambos para crear funciones muy versﾃ｡tiles.

    Uso de *args (argumentos posicionales variables)

    Cuando usamos *args en la definiciﾃｳn de una funciﾃｳn, estamos diciendo que esa funciﾃｳn puede recibir muchos valores sin nombre, que serﾃ｡n agrupados en una tupla. Esto es ﾃｺtil cuando queremos operar con una lista de nﾃｺmeros, textos u otros elementos sin saber cuﾃ｡ntos serﾃ｡n.

    Se puede ver mﾃ｡s claro en los siguientes ejemplos:

    Ejemplo 1: Sumar varios nﾃｺmeros pasados como parﾃ｡metros

    def sumar_todos(*numeros):
    """Suma todos los nﾃｺmeros que se pasen como argumentos."""
    return sum(numeros)

print(sumar_todos(2, 4, 6))       # 12
print(sumar_todos(1, 3, 5, 7, 9)) # 25

    Ejemplo mostramos los parﾃ｡metros pasados

    def mostrar_nombres(*nombres):
    print("Lista de nombres:")
    for nombre in nombres:
        print("-", nombre)

mostrar_nombres("Lucﾃｭa", "Carlos", "Marta")
mostrar_nombres("Sergio")

    En ambos casos, la funciﾃｳn puede recibir uno, tres o veinte argumentos sin cambiar su definiciﾃｳn.

    Uso de **kwargs (argumentos nombrados variables)

    Cuando usamos **kwargs, la funciﾃｳn puede recibir muchos argumentos con nombre, que serﾃ｡n agrupados en un diccionario. Esto es ﾃｺtil cuando queremos que el usuario pueda especificar informaciﾃｳn sin seguir un orden fijo.

    De nuevo comprenderemos mejor su uso con un par de ejemplos. Observar como los parﾃ｡metros pasados deben tener formato de diccionario:

    Mostrar informaciﾃｳn de usuario

    def mostrar_info(**datos):
    for clave, valor in datos.items():
        print(f"{clave}: {valor}")

mostrar_info(nombre="Sergio", edad=25, ciudad="Alberic")

    Observar cﾃｳmo se han recuperado tanto la clave como el valor de cada tupla del diccionario

    Veamos de nuevo otro ejemplo similar

    Ejemplo de lectura de configuraciﾃｳn

    def configurar(**opciones):
    print("Configuraciﾃｳn aplicada:")
    for clave, valor in opciones.items():
        print(f"{clave} = {valor}")

    En estos ejemplos, los argumentos se pasan como pares clave-valor, y la funciﾃｳn los procesa como un diccionario.

    Combinaciﾃｳn de parﾃ｡metros fijos, *args y **kwargs

    Tambiﾃｩn se pueden combinar parﾃ｡metros normales con *args y **kwargs en una misma funciﾃｳn. El orden debe ser:

    1. Parﾃ｡metros normales
    2. *args
    3. **kwargs

    Esto permite que la funciﾃｳn tenga una parte fija, una parte variable de datos, y una parte flexible de configuraciﾃｳn.

    De nuevo veamos un par de ejemplos:

    Procesar datos con nombre y valores

    print("Nombre:", nombre)
    print("Valores:", valores)
    print("Opciones:", opciones)

    Observar en este ejemplo como el primer parﾃ｡metro de la invocaciﾃｳn de la funciﾃｳn pertenece al parﾃ｡metro fijo y el resto de parﾃ｡metros simple se convierten en una lista para los parﾃ｡metros posicionales y los parﾃ｡metros pasados como tuplas clave-valor se convierten en los parﾃ｡metros finales en modo diccionario.

    Veamos otro ejemplo similar

    Enviar mensaje personalizado

    def enviar_mensaje(destinatario, *lineas, **firma):

    Estas funciones permiten construir estructuras muy flexibles, ideales para proyectos reales donde los datos pueden variar segﾃｺn el contexto.

    洫ｩ Ejemplo y ejercicio de uso

    En el siguiente ejemplo se muestra cﾃｳmo definir una funciﾃｳn que acepta una cantidad variable de argumentos posicionales (*args) y nombrados (**kwargs). La funciﾃｳn simula el registro de un pedido, donde se pueden aﾃｱadir productos sin lﾃｭmite y configurar opciones adicionales como envﾃｭo o mﾃｩtodo de pago.

    Este ejemplo muestra cﾃｳmo se pueden combinar argumentos fijos, mﾃｺltiples valores y opciones flexibles en una sola funciﾃｳn.

    ﾃ［bito de variables

    Tal y como sucede en otros lenguajes, debemos diferenciar entre variables locales y globales. Las variables definidas dentro de una funciﾃｳn son locales y no afectan al resto del programa. Las variables globales pueden ser accedidas desde cualquier parte, pero deben usarse con cuidado para evitar conflictos.

    Variables locales

    Una variable local es aquella que se define dentro de una funciﾃｳn y solo existe mientras esa funciﾃｳn se estﾃ｡ ejecutando. No puede ser accedida desde fuera.

    Error en uso de variable local

    def saludar():
    mensaje = "Hola desde dentro de la funciﾃｳn"
    print(mensaje)

saludar()
print(mensaje)  # 笶 Esto da error: 'mensaje' no estﾃ｡ definida fuera de la funciﾃｳn

    En este ejemplo, mensaje solo existe dentro de saludar(). Al intentar usarla fuera, Python no la reconoce.

    Variables globales

    Una variable global se define fuera de cualquier funciﾃｳn y puede ser utilizada dentro de funciones, pero solo para leer su valor. Si se intenta modificarla directamente dentro de una funciﾃｳn, Python la tratarﾃ｡ como una nueva variable local, a menos que se indique lo contrario.

    Uso correcto de variable global

    contador = 0  # Variable global

def mostrar_contador():
    print("Contador global:", contador)

mostrar_contador()  # 笨 Se puede leer sin problema

    Aquﾃｭ, contador es global y puede ser leﾃｭdo desde dentro de la funciﾃｳn sin necesidad de hacer nada especial.

    Variables con el mismo nombre: local vs. global

    Si dentro de una funciﾃｳn se define una variable con el mismo nombre que una global, la versiﾃｳn local oculta la global dentro de ese ﾃ｡mbito.

    Ejemplo de ﾃ｡mbito de uso de variables local y global

    mensaje = "Hola desde fuera"

def cambiar_mensaje():
    mensaje = "Hola desde dentro"
    print("Dentro de la funciﾃｳn:", mensaje)

cambiar_mensaje()
print("Fuera de la funciﾃｳn:", mensaje)

    En este caso, aunque ambas variables se llaman mensaje, son distintas. La funciﾃｳn usa su propia versiﾃｳn local y no modifica la global.

    Modificar una variable global desde una funciﾃｳn

    Si realmente se quiere modificar una variable global desde dentro de una funciﾃｳn, hay que usar la palabra clave global.

    Ejemplo de modificaciﾃｳn de variable global desde funciﾃｳn

    contador = 0

def incrementar():
    global contador
    contador += 1

incrementar()
print("Contador despuﾃｩs de la funciﾃｳn:", contador)  # 1

    Aquﾃｭ, al declarar global contador, le decimos a Python que queremos usar la variable global y no crear una nueva local. Esto permite modificar su valor desde dentro de la funciﾃｳn.

    Manejo de excepciones en Python

    En Python, una excepciﾃｳn es un tipo de error que ocurre mientras el programa se estﾃ｡ ejecutando. A diferencia de los errores de sintaxis, que impiden que el programa arranque, las excepciones aparecen cuando algo inesperado sucede durante la ejecuciﾃｳn: por ejemplo, dividir entre cero, acceder a una posiciﾃｳn inexistente en una lista o convertir una cadena que no es un nﾃｺmero con int().

    Cuando ocurre una excepciﾃｳn y no se gestiona, el programa se detiene y muestra un mensaje de error. Para evitar esto, Python permite capturar y manejar excepciones usando las instrucciones try, except, finally y raise. Esto permite que el programa continﾃｺe funcionando de forma controlada, incluso si algo falla.

    El manejo de excepciones es especialmente ﾃｺtil cuando se trabaja con entrada de datos, operaciones matemﾃ｡ticas, acceso a archivos o cualquier situaciﾃｳn en la que el comportamiento del usuario o del entorno pueda provocar errores. Saber identificar y tratar estos errores es fundamental para escribir programas robustos y profesionales.

    Uso de try except

    Veamos un ejemplo sencillo de captura de excepciones.

    Captura de excepciﾃｳn

    try:
    x = int("abc")
except:
    print("Algo saliﾃｳ mal")  # Captura cualquier excepciﾃｳn

    En este ejemplo, se captura el error aunque no se especifique que es un . Sin embargo, si se quiere saber quﾃｩ tipo de error ocurriﾃｳ, se puede capturar el objeto de la excepciﾃｳn:

    Captura de excepciﾃｳn obteniendo el mensaje de la excepciﾃｳn

    try:
    x = int("abc")
except Exception as e:
    print("Error:", e)

    Esto permite mostrar el mensaje del error sin detener el programa.

    Pero tambiﾃｩn podemos esperar capturar un excepciﾃｳn en concreto, por ejemplo la tﾃｭpica de divisiﾃｳn por cero.

    Ejemplo de captura de excepciﾃｳn de divisiﾃｳn entre cero

    def dividir(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("Error: No se puede dividir entre cero.")
